Git & GitHub Practice for Freshers — Step‑by‑Step (HTML files)

This single page guides you through a gentle, hands‑on practice using Git locally and GitHub remote, working with small HTML files. Follow each step on your computer and copy commands into your terminal. Answers and full command lists are given in the Solutions section at the end.

Prerequisites

  • A computer with Git installed. (Check with git --version)
  • A GitHub account (https://github.com) — free accounts are fine.
  • A code editor (VS Code, Sublime, Notepad++) or simple text editor.
  • Basic terminal/command‑line familiarity.

Step 0 — Quick checks

git --version
whoami  # or check your OS username

Step 1 — Create a practice folder and initialize Git

Open terminal and run:

mkdir git-practice-html
cd git-practice-html
git init

Git created a .git folder. Now configure your identity if not set:

git config --global user.name "Your Name"
git config --global user.email "you@example.com"

Step 2 — Add your first HTML file (index.html)

Create a file named index.html with this content:

<!doctype html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <title>My Git Practice</title>
</head>
<body>
  <h1>Hello, Git!</h1>
  <p>This is my first file in the git-practice repository.</p>
</body>
</html>

Save the file and then add & commit it:

git status
git add index.html
git commit -m "Add index.html: Hello Git"

Step 3 — Create another HTML page and link it

Create about.html:

<!doctype html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <title>About</title>
</head>
<body>
  <h1>About This Practice</h1>
  <p>A short description about the practice.</p>
  <a href="index.html">Back to Home</a>
</body>
</html>

Add a link from index.html to about.html, then:

git add about.html index.html
git commit -m "Add about.html and link from index"

Step 4 — Branching practice

Create a new branch to add a contact page:

git branch add-contact
git checkout add-contact
# or shorthand: git checkout -b add-contact

Create contact.html with a small contact form placeholder, then:

git add contact.html
git commit -m "Add contact.html on add-contact branch"

Step 5 — Merge branch into main

git checkout main
git merge add-contact

If merge is a fast-forward, it will integrate. If there is a conflict, Git will mark files; resolve the conflict in the editor, then:

git add <resolved-file>
git commit -m "Resolve merge conflict"

Step 6 — Create a remote GitHub repo and push

  1. On GitHub website, click New repository. Name it git-practice-html. Keep it public (or private) and do NOT initialize with README (we'll push existing repo).
  2. GitHub will show commands to connect your local repo. Use these (example):
git remote add origin https://github.com/your-username/git-practice-html.git
git branch -M main
git push -u origin main

After this, your HTML files are on GitHub.

Step 7 — Collaborating: clone, edit, pull request

To simulate a collaborator, clone your repo into a new folder:

cd ..
git clone https://github.com/your-username/git-practice-html.git git-practice-copy
cd git-practice-copy

Make a change in index.html (e.g., add a paragraph), commit and push on a new branch:

git checkout -b improve-index
# edit index.html
git add index.html
git commit -m "Improve index content"
git push -u origin improve-index

Then on GitHub open a Pull Request from improve-index into main. Accept it to merge.

Step 8 — GitHub Pages (host your HTML)

To publish your site from the main branch:

  1. On GitHub repo page, go to Settings > Pages.
  2. Choose branch: main and folder: / (root). Save.

GitHub will provide a URL like https://your-username.github.io/git-practice-html/. Visit it to see your HTML pages live.

Common Git Commands — Quick Reference

git status        # see changed files
git add      # stage changes
git commit -m "msg"   # record changes
git log --oneline  # view commits
git branch         # list branches
git checkout    # switch branch
git checkout -b # create & switch to branch
git merge       # merge branch
git remote -v      # view remotes
git push           # push changes
git pull           # fetch & merge remote changes

Practice Exercises

  1. Create a new branch called feature-navbar and add a simple navigation bar to index.html, commit and merge it back to main.
  2. Simulate a conflict: Two clones edit the same line in about.html. Push both branches and resolve the merge conflict on one of them.
  3. Make a small enhancement, open a Pull Request, then merge on GitHub (use GitHub UI).
  4. Enable GitHub Pages and share your live URL.
Mini assignment: Build a 3‑page mini site (Home, About, Contact), host with GitHub Pages, and share the repo URL and Pages URL.

Solutions & Commands (cheat sheet)

Below are the exact commands you can copy & paste when doing the steps above (replace your-username):

# create repo locally
mkdir git-practice-html
cd git-practice-html
git init
# set identity (only once)
git config --global user.name "Your Name"
git config --global user.email "you@example.com"

# add files
# (create index.html, about.html, contact.html with editor)

git add .
git commit -m "Initial site: index, about, contact"

# create branch and add contact page
git checkout -b add-contact
# (create contact.html)
git add contact.html
git commit -m "Add contact page"

# merge back
git checkout main
git merge add-contact

# connect to GitHub (after creating remote repo on github.com)
git remote add origin https://github.com/your-username/git-practice-html.git
git branch -M main
git push -u origin main

# create a feature branch, push, open PR
git checkout -b feature-navbar
# edit files
git add .
git commit -m "Add navbar"
git push -u origin feature-navbar

Troubleshooting tips

  • Permission denied on push: You may need to set up SSH keys or use a personal access token for HTTPS pushes. GitHub docs explain both options.
  • Merge conflicts: Open the file, find <<<<<<< HEAD markers, edit to keep desired content, then git add and git commit.
  • Forgot to add a file: Use git add and git commit --amend to update last commit (careful if already pushed).
